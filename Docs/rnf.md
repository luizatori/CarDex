# RNF - Requisitos Não Funcionais

## 1. Introdução

Este documento descreve os Requisitos Não Funcionais do aplicativo.

---

## 2. Lista de Requisitos Não Funcionais

### RNF01 – Desempenho
O sistema deve processar a identificação do veículo em até 5 segundos após o envio da imagem.

---

### RNF02 – Segurança
- As senhas devem ser armazenadas de forma criptografada.
- As imagens devem ter as placas censuradas antes de qualquer armazenamento.

---

### RNF03 – Usabilidade
- A interface deve ser intuitiva e adequada para dispositivos móveis.
- O fluxo de cadastro e login deve ser simples e direto.

---

### RNF04 – Compatibilidade
O aplicativo deve ser compatível com:
- Android (versão mínima definida pelo projeto);
- iOS (versão mínima definida pelo projeto).

---

### RNF05 – Disponibilidade
O sistema deve possuir disponibilidade mínima de 95% ao mês.

---

### RNF06 – Escalabilidade
O sistema deve suportar crescimento progressivo da base de usuários sem perda significativa de desempenho.

---

### RNF07 – Manutenibilidade
O código deve seguir boas práticas de desenvolvimento e arquitetura modular.

---

## 3. Padrões e Tecnologias

- Flutter para desenvolvimento mobile;
- API para processamento de IA;
- Banco de dados a ser definido no projeto.
